This page will show you the output of the JSON-LD API.  If you are calling the API from JavaScript then you will want to include the jsonld-helper.js library and use it to get the single Javascript object containing your data (jsonldHelper.getItem). Note that the "over the wire" JSON-LD data is flattened out, with all object-to-object references represented by URI's.  This allows for serialization of the RDF graph even when we have circular references, but having your code work the flat JSOND-LD directly is cumbersome so it is recommended to use the jsonldHelper.getItem method.  View the source for this page to see how all of this is done.

If you are not calling the JSON-LD API from JavaScript and are instead using server side code, you will need to work with the raw JSON-LD.  You should look at jsonld-helper.js and port the jsonldHelper.getItem algorithm to whatever server side language you are using.  While this is not necessary, it will likely make your life much easier!

Note: Gadgets should not use this API directly, but should instead use the osapi.rdf methods which provide the same functionality (and more), all within the OpenSocial framework.  Please see the ORNG documentation for details.
